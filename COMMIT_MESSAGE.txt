feat(config,report,main): Add preset system, config file support, and Kafka streaming

## Summary
Major restructuring and feature expansion to support Streamlit UI integration and 
enterprise operations. Maintains full backward compatibility while introducing 
preset-based execution, YAML/JSON configuration files, Kafka event streaming, 
and granular rule control.

## Features
- feat(config): Add preset system (safe, balanced, deep, handover, offboarding)
- feat(config): Support YAML/JSON configuration files via --config
- feat(config): Add --server-type nginx|apache|manual with validation
- feat(config): Implement CLI + file + preset merge strategy
- feat(report): Add Config and ActiveRules fields to report
- feat(report): Kafka event streaming with franz-go library
- feat(main): Implement rule-level enable/disable control
- feat(main): Add organized help output with option groups
- feat(main): Auto-detect server-type from input flags
- feat(main): Log final configuration for transparency

## Config Changes
- Added KafkaConfig struct with 9 TLS/SASL options
- Extended Config struct with 6 new fields
- New stringSliceFlag type for comma-separated list support
- Nested Rules struct in Config for YAML/JSON format

## Code Changes
- internal/config/config.go
  * Rewrote MustParseFlags() with 3-stage merge (file→preset→CLI)
  * Added LoadFromFile(), mergeConfig(), scanArgValue(), applyPreset()
  * Added mergeStringSlices() helper
  * Supports YAML (gopkg.in/yaml.v3) and JSON (encoding/json)

- cmd/dmz_webroot_scanner/main.go
  * Separated makeRuleSet() factory for rule composition
  * Added validateInputCombination() for input validation
  * Added printFlagGroup() for organized help
  * Auto-detect and log final configuration
  * Record active rules in report
  * Call SendToKafka() after scan (non-blocking on error)

- internal/report/model.go
  * Added Config interface{} field for applied settings
  * Added ActiveRules []string for transparency

- internal/report/kafka.go (new file)
  * KafkaEvent struct for summarized event schema
  * makeEvents() for conversion from Report
  * SendToKafka() with TLS support and SASL stub
